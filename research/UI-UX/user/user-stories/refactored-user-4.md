# User Story 4: Team-Specific AI Guidance

**As a Team Lead,** I want to define specific "steering rules" for my workspace, so that the AI's suggestions align with our specific team's coding standards and labeling conventions.

### Step-by-Step Experience:
1. **Settings Access:** User navigates to the "Team Settings" -> "AI Configurations" page.
2. **Rule Definition:** User adds a steering rule: *"Always tag issues related to the UI with the 'Frontend' label and assign them to the 'UI-Squad' team."*
3. **Test Run:** User creates a dummy issue titled "Fix header padding."
4. **Observation:** User watches the AI process the issue and confirms it correctly followed the new rule by applying the 'Frontend' label.
5. **Refinement:** User realizes the AI is being too aggressive with labels and updates the rule to be more specific.
6. **Persistence:** The updated "Human Direction" is now the default behavior for all future AI actions in that specific workspace.
