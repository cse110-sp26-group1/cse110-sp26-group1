# User Story 2: AI Workload and Budget Visibility

## Story

As a project lead using the Agent Issue Tracker, I want to see token usage, estimated cost, and agent activity connected to each issue so that I can manage team resources and decide when AI assistance is worth using.

## Why This Matters

An AI-focused issue tracker needs to show more than task status. Teams also need visibility into how much AI work is being requested, how expensive it is, and which issues are consuming the most time or budget. This helps the team plan responsibly without burying the information in reports.

## Acceptance Criteria

- Each issue can display related AI activity, including token usage, estimated cost, and latest agent update.
- The dashboard summarizes total token usage and estimated budget used across active issues.
- Users can sort or filter issues by AI usage, cost, or agent status.
- Warnings appear when an issue or project is close to a budget limit.
- Resource information is presented in a clear, compact way that does not distract from normal issue tracking.

## UI/UX Notes

- Use simple charts, labels, or badges to make usage data easy to scan.
- Group AI usage details near issue metadata so users can compare effort, priority, and cost together.
- Keep the dashboard readable with strong hierarchy and consistent spacing.
