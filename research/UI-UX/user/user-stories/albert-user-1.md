# User Story 1: Fast Issue Triage

## Story

As a developer using the Agent Issue Tracker, I want to quickly scan, filter, and update issues from the main issue list so that I can decide what needs attention without opening every issue individually.

## Why This Matters

The issue tracker should support AI-speed planning with less friction than heavier tools. A clear issue list with strong visual hierarchy, accessible filters, and direct status controls helps users move through triage quickly while still understanding priority, tags, owners, and issue state.

## Acceptance Criteria

- The issue list clearly displays each issue's title, status, priority, tags, date, and assigned owner.
- Users can filter or search issues by status, priority, tag, and text.
- Users can update an issue's status directly from the issue list.
- Important statuses and priorities use consistent visual indicators that are readable and accessible.
- Clicking an issue card or row opens the full issue details for editing.

## UI/UX Notes

- Use color to distinguish issue status and priority, but do not rely on color alone.
- Keep search, filters, and common issue actions close together.
- Make clickable areas visually clear so users do not have to guess where to interact.
