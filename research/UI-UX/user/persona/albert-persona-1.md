# Persona 1: Busy Student Developer

## Background

**Name:** Maya
**Role:** Student developer and project lead  
**Experience Level:** Intermediate  
**Context:** Maya is working on a software engineering team project where issues are created, updated, and assigned quickly. Her team uses AI tools to help write code, summarize tasks, and investigate bugs, but she still needs to keep the project organized for human teammates.

## Goals

- Quickly understand which issues need attention first.
- Keep the team's issue list organized without spending too much time on management.
- Track issue status, priority, owner, and tags from one clear view.
- Understand how much AI assistance is being used on each task.
- Avoid going over the team's AI token or budget limits.

## Frustrations

- Issue trackers can feel crowded when too many details are shown at once.
- Updating issue status is annoying when it requires opening every individual issue.
- It is hard to know which issues are taking the most AI time, tokens, or budget.
- Inconsistent tags, colors, and layouts make scanning issues slower.
- Important information can be missed if the interface does not have clear visual hierarchy.

## Needs

- A clean dashboard that makes priority, status, and ownership easy to scan.
- Search and filters for status, priority, tags, and AI usage.
- Direct controls for changing issue status from the main issue list.
- Clear visual indicators that are accessible and do not rely only on color.
- Compact AI usage summaries that show token usage, cost, and recent agent activity.

## Scenario

Maya opens the Agent Issue Tracker before a team meeting. She needs to review open issues, identify blocked tasks, check which issues are using the most AI resources, and update task statuses before assigning new work. She wants to do this quickly from the main dashboard instead of clicking through every issue one by one.

## Quote

"I want to see what needs work, what is blocked, and how much AI help we are using without digging through every issue."
